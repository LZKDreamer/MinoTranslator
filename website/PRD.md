# PRD: Mino Translator官网

> **产品名称**: Mino 翻译/Mino Translator
> **站点域名**: www.zaymino.com（要加二级域名）  
> **站点类型**: Chrome 翻译扩展官网  
> **首版结构**: 单页官网  
> **默认语言**: 简体中文  
> **支持语言**: 简体中文、English  
> **部署方式**: Vercel  
> **可选存储**: Supabase  

---

## 1. 背景

Zaymino 是一款 Chrome MV3 翻译扩展，核心能力是 YouTube 字幕实时翻译，辅助能力是网页划词翻译。插件当前未上架 Chrome Web Store，因此官网需要承担产品介绍、信任建立、ZIP 安装包下载、安装教学和常见问题解答。

官网第一版不做注册登录、不做用户后台、不做复杂内容站。目标是用一个清晰的单页完成从理解产品价值到下载插件的路径，并为后续 SEO、版本更新和 Chrome Web Store 上架预留扩展空间。

---

## 2. 目标

### 2.1 业务目标

1. 让首次访问用户快速理解 Zaymino 的核心价值：在 YouTube 上稳定显示双语字幕，并在网页中快速划词翻译。
2. 让用户明确知道当前插件尚未上架 Chrome Web Store，首版通过 ZIP 安装包下载和开发者模式安装。
3. 降低手动安装门槛，用户能按照官网步骤完成下载、解压、加载已解压扩展、填写 API Key。
4. 建立一个可持续维护的官网基础，后续可扩展下载统计、版本管理、独立教程页、SEO 功能页。

### 2.2 用户目标

1. 在 30 秒内判断插件是否适合自己。
2. 在 2 分钟内理解安装方式和必要配置。
3. 清楚知道自己需要准备 OpenAI-compatible API Key。
4. 了解插件如何处理 API Key、翻译文本和隐私数据。

### 2.3 非目标

1. 不提供注册、登录、用户中心。
2. 不提供在线翻译 Web App。
3. 不做 Chrome Web Store 上架页替代文案，如“从商店安装”。
4. 不在第一版实现博客、文档中心、多级 SEO 内容站。

## 3. 用户与场景

### 3.1 主要用户

观看外语 YouTube 视频、需要实时字幕翻译的用户。他们关注字幕同步、可读性、翻译质量、安装和配置成本。

### 3.2 次要用户

日常浏览外文网页、需要快速划词翻译的用户。他们关注是否打扰阅读、是否响应迅速、是否支持常见网页。

### 3.3 核心场景

| 场景 | 用户问题 | 官网需要回答 |
|---|---|---|
| 看外语 YouTube 视频 | 能不能把字幕翻译出来并跟随视频播放？ | 展示双语字幕覆盖播放器的核心场景 |
| 阅读外文网页 | 能不能选中文本后快速翻译？ | 展示划词图标和翻译气泡流程 |
| 手动安装插件 | 没有上架 Chrome 商店怎么安装？ | 提供 ZIP 下载和开发者模式安装步骤 |
| 配置模型 | 是否需要账号或 API Key？ | 说明无需官网账号，但需要用户自备 OpenAI-compatible API Key |
| 隐私顾虑 | API Key 和文本会不会上传到官网？ | 说明官网无登录，插件请求发送到用户配置的模型服务 |

## 4. 产品定位

### 4.1 一句话定位

Zaymino 是一个为 YouTube 视频和网页阅读设计的 Chrome 翻译扩展，让双语字幕和划词翻译在需要时出现，不需要时退到后台。

### 4.2 核心卖点

1. **YouTube 双语字幕翻译**: 自动获取字幕，AI 翻译后覆盖显示在播放器上。
2. **网页划词翻译**: 任意网页选中文本后点击图标，查看原文和译文。
3. **自带模型选择空间**: 支持 OpenAI-compatible API，用户可配置自己的模型服务。
4. **克制不打扰**: 工具只在需要时出现，视觉和交互不抢视频与阅读内容。
5. **可配置字幕显示**: 支持原文、双语、译文模式，以及字幕字号、位置、颜色、背景透明度设置。

### 4.3 品牌性格

简洁、高效、友好。官网应避免“炫酷 AI 工具”表达，强调可靠、清楚、轻量、可控。

## 6. 页面功能需求

### 6.1 顶部导航

#### 内容

1. 左侧展示 Zaymino logo 和产品名。
2. 右侧展示锚点导航、语言切换、下载按钮。
3. 移动端使用折叠菜单。

#### 交互

1. 点击锚点平滑滚动到对应模块。
2. 当前语言可切换为中文或英文。
3. 下载按钮滚动到下载模块，或直接触发下载动作，具体由实现阶段根据下载资源状态决定。

### 6.2 Hero 首屏

#### 目标

让用户第一眼知道这是一个 Chrome 翻译扩展，并理解最主要场景是 YouTube 字幕翻译。

#### 内容

1. 产品名: Zaymino。
2. 主标题: 围绕 YouTube 字幕翻译和网页划词翻译表达。
3. 副标题: 说明支持 OpenAI-compatible API，用户自配模型，当前通过 ZIP 安装包提供下载。
4. 主 CTA: 下载 ZIP。
5. 次 CTA: 查看安装步骤。
6. 明确状态提示: 暂未上架 Chrome Web Store。

#### 视觉

1. 首屏必须出现产品实际能力信号，例如播放器中的双语字幕示意、插件弹窗示意或划词气泡示意。
2. 不使用抽象 AI 渐变背景作为主视觉。
3. 不把所有内容包在大卡片里，优先使用清晰的布局和真实产品场景。

### 6.3 产品场景预览

#### 内容

展示两个核心场景：

1. YouTube 播放器上显示原文和译文字幕。
2. 网页选中文本后出现翻译按钮和译文气泡。

#### 约束

1. 优先使用产品真实截图或接近真实 UI 的静态预览。
2. 如果首版没有截图，可以使用明确标注的产品示意图，但不伪装成真实截图。
3. logo 使用 `D:\codeproject\translator\extension\icons` 下的现有素材。

### 6.4 核心功能

#### 功能点

1. YouTube 字幕实时翻译。
2. 双语、原文、译文三种字幕显示模式。
3. 字幕外观可配置。
4. 网页划词翻译。
5. 支持 OpenAI-compatible API。
6. 翻译缓存和失败重试。

#### 展示方式

功能说明应短而具体，避免泛泛表达“AI 驱动”“提升效率”。每个功能应对应一个用户场景。

### 6.5 工作方式

用 3 到 4 步解释用户旅程：

1. 下载 ZIP 安装包。
2. 在 Chrome 扩展管理页面加载已解压扩展。
3. 填写 OpenAI-compatible API Key。
4. 打开 YouTube 或网页开始翻译。

### 6.6 ZIP 下载

#### 内容

1. 下载按钮: 下载 Zaymino ZIP 安装包。
2. 当前版本号。
3. 更新时间。
4. 文件大小。
5. 安装方式摘要。
6. Chrome Web Store 状态: 暂未上架。

#### 数据来源

首版可以静态配置下载信息。后续如使用 Supabase，建议存储：

1. `version`: 版本号。
2. `file_url`: ZIP 文件地址。
3. `file_size`: 文件大小。
4. `sha256`: 文件校验值。
5. `release_notes`: 更新说明。
6. `published_at`: 发布时间。
7. `download_count`: 下载次数。

#### 下载行为

1. 点击下载后应下载 ZIP 文件。
2. 如果下载文件暂未配置，按钮应进入明确的不可用状态或显示“即将开放下载”，不能静默失败。
3. 下载区域需要提示用户：下载后请解压，并通过 Chrome 开发者模式加载已解压扩展。

### 6.7 安装步骤

安装步骤必须覆盖：

1. 下载 ZIP。
2. 解压到本地文件夹。
3. 打开 `chrome://extensions`。
4. 启用开发者模式。
5. 点击“加载已解压的扩展程序”。
6. 选择解压后的插件目录。
7. 打开扩展设置，填写 API URL、API Key、Model ID。

需要明确说明当前不是 Chrome Web Store 安装流程。

### 6.8 API Key 配置说明

#### 内容

1. Zaymino 支持 OpenAI-compatible API。
2. 用户需要自备 API Key。
3. 默认可展示 Agnes AI / DeepSeek 等预设概念，但不应承诺官网提供 API Key。
4. API Key 存储在浏览器扩展本地或 Chrome storage 中，官网不提供账号系统。

### 6.9 FAQ

首版 FAQ 至少包含：

1. Zaymino 是否已经上架 Chrome Web Store？
2. 为什么需要开发者模式安装？
3. 是否需要自备 API Key？
4. 支持哪些语言？
5. 支持哪些网站？
6. YouTube 没有字幕时还能翻译吗？
7. API Key 会发送到官网吗？
8. 如何更新插件？

FAQ 需要支持结构化数据，便于 SEO。

### 6.10 隐私与信任说明

#### 内容

1. 官网无注册登录功能。
2. 官网不处理用户翻译内容。
3. 插件翻译请求发送到用户配置的 OpenAI-compatible API 服务。
4. API Key 由插件存储在浏览器环境中。
5. 提供隐私政策入口。首版可以链接到同页锚点，后续可扩展为独立页面。

## 8. 视觉与体验要求

### 8.1 Design Read

这是一个 Chrome 扩展产品官网，面向需要 YouTube 字幕翻译和网页划词翻译的个人用户。视觉语言应克制、可信、工具感明确，偏 DeepL 式蓝白极简，但需要保留更强的产品截图和安装引导。

### 8.2 设计参数

参考 taste-skill，建议：

1. `DESIGN_VARIANCE: 5`
2. `MOTION_INTENSITY: 3`
3. `VISUAL_DENSITY: 4`

### 8.3 视觉原则

1. 主色建议使用蓝色系，但避免大面积高饱和渐变。
2. 背景以白色或接近白色为主，使用清晰分区和留白。
3. 卡片只用于功能项、FAQ、下载信息等有明确边界的内容，不做卡片套卡片。
4. 圆角控制在 8px 左右，按钮可以更圆但需保持规则一致。
5. 图标库使用 lucide，符合用户指定来源。
6. 不使用 emoji 作为功能图标。
7. 首页需要使用真实 logo 资源。
8. 动效以轻微滚动出现、按钮反馈、锚点滚动为主，不做大幅视差或复杂动画。

### 8.4 可访问性

1. 正文文本对比度不低于 WCAG AA。
2. 按钮和链接必须有清晰 hover、focus、active 状态。
3. 下载、安装、FAQ 等关键内容在移动端不可折叠到难以发现的位置。
4. 所有图片和产品示意图需要 alt 文案，且 alt 文案进入 i18n。

## 9. 技术与部署要求

### 9.1 前端

推荐使用 React / Next.js 构建，便于 Vercel 部署、SEO、i18n 和后续页面扩展。具体技术选型在实现阶段确认。

### 9.2 部署

1. 前端部署到 Vercel。
2. 生产域名为 `www.zaymino.com`。
3. 需要配置 HTTPS。
4. 需要提供基础 SEO meta、Open Graph、favicon、robots.txt、sitemap.xml。

### 9.3 Supabase

Supabase 为可选能力，不是首版硬依赖。

可用于：

1. 存储插件版本信息。
2. 存储 ZIP 下载地址。
3. 记录下载次数。
4. 后续支持订阅更新通知。

首版如果没有后端，可使用静态 JSON 或环境变量配置下载信息。

### 9.4 资源

1. logo 和图标资源来自 `D:\codeproject\translator\extension\icons`。
2. 插件 ZIP 文件由发布流程产出，官网只负责展示和下载。
3. 不从 Chrome Web Store 拉取安装状态，因为当前尚未上架。

## 13. 风险与约束

1. ZIP 手动安装比商店安装门槛更高，安装步骤必须清楚。
2. 未上架 Chrome Web Store 会影响用户信任，需要通过隐私说明、版本信息、文件校验值和清晰文案降低疑虑。
3. 插件依赖用户自备 API Key，官网不能暗示无需配置即可翻译。
4. 如果没有真实产品截图，首版示意图需要避免误导。
5. 中英文文案需要保持信息一致，避免中文更新后英文遗漏。
